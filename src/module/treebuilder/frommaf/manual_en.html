Tree Builder: from MAF

Reconstruct phylogenetic trees from UCSC MAF (Multiple Alignment Format) blocks.

1. Quick Start

  1. Select the folder that contains your MAF (or MAF.GZ) files
  2. Adjust distance and tree reconstruction parameters if needed
  3. Click Execute to see the results

2. Overview

This module reconstructs a phylogenetic tree from UCSC MAF (Multiple Alignment Format) blocks. It parses all MAF blocks, computes pairwise evolutionary distances across blocks, then builds a distance-based tree (Neighbor-Joining, Swift Neighbor-Joining, or UPGMA). The final tree is rooted by the mid-point method and opened in the Modern Tree Viewer.

3. Input Format

Input should be a directory containing MAF files. The module loads all files ending with .maf (or .maf.gz if Is Gzip is checked).

3.1 MAF Format Example

a score=...
s hg38.chr1  ... ACTG--A...
s panTro4.chr1 ... ACTG--A...
s ponAbe2.chr1 ... ACTG--A...

3.2 Format Rules

  • Each file must be standard MAF with blocks separated by a blank line
  • Sequence lines start with s
  • Only blocks that contain all required assemblies are used
  • In the current GUI, the assemblies are fixed to panTro4, hg38, and ponAbe2

4. Parameters

Parameter Description
Input Folder path containing MAF files
Is Gzip Check if files end with .maf.gz
Substitution model / Gaps & missing data Choose the evolutionary distance model (Model/Method) and deletion strategy (Complete/Pairwise/Partial deletion)
Variance estimation Set to Bootstrap method and choose No. of Bootstrap replications to obtain bootstrap supports
Distance based method Neighbor-Joining (default), Swift Neighbor-Joining, or UPGMA
Genetic code / IUPAC treatment Controls how ambiguous bases are treated when distances are computed from blocks
