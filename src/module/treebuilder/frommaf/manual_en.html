Tree builder: from MAF

Task to resolve

This module reconstructs a phylogenetic tree from UCSC MAF (Multiple Alignment Format) blocks. It parses all MAF blocks, computes pairwise evolutionary distances across blocks, then builds a distance‑based tree (Neighbor‑Joining, Swift Neighbor‑Joining, or UPGMA). The final tree is rooted by the mid‑point method and opened in the Modern Tree Viewer.

Input format

Input should be a directory containing MAF files. The module loads all files ending with .maf (or .maf.gz if Is Gzip is checked). Each file must be standard MAF with blocks separated by a blank line and sequence lines starting with s:

a score=...
s hg38.chr1  ... ACTG--A...
s panTro4.chr1 ... ACTG--A...
s ponAbe2.chr1 ... ACTG--A...

Only blocks that contain all required assemblies are used. In the current GUI, the assemblies are fixed to panTro4, hg38, and ponAbe2.

Parameters

  1. Input: folder path containing MAF files.
  2. Is Gzip: check if files end with .maf.gz.
  3. Substitution model / Gaps & missing data: choose the evolutionary distance model (Model/Method) and deletion strategy (Complete/Pairwise/Partial deletion).
  4. Variance estimation: set Variance Estimation Method to Bootstrap method and choose No. of Bootstrap replications to obtain bootstrap supports.
  5. Distance based method: Neighbor‑Joining (default), Swift Neighbor‑Joining, or UPGMA.
  6. Genetic code / IUPAC treatment: controls how ambiguous bases are treated when distances are computed from blocks.

Quick start

Any actions are straightforward. Please follow the GUI instructions.

  1. Select the folder that contains your MAF (or MAF.GZ) files.
  2. Adjust distance and tree reconstruction parameters if needed.
  3. Click the Execute button to see the results.
