Interactive visualization and operation of phylogenetic trees. Supports multiple tree layouts, color configurations, and graphical annotations with high-quality vector graphics export.

Quick start

Any actions are straightforward. Please follow the GUI instructions.

  1. Click the Example button to see the input data.
  2. Click the Execute button to see the results.

  2. Control panel buttons, divided into four categories:
    1. First category: Gesture buttons
    2. Second category: Normal buttons
    3. Third category: Counting gesture buttons
    4. Fourth category: Swing Toggle buttons
  3. Highly recommended is this Toggle button with an Apple desktop style, which is used in the Tree layout and Creative mode panels.
  4. VOICM solves the parameter explosion issue:
    1. Bookmarks that can be recorded, support free operations, and can be imported and exported.
    2. All parameters can be set in the input panel.

Input Parameters Reference

Tree Input Methods

There are three ways to input a phylogenetic tree (in priority order):

1. input.nwk.string

Description: Direct input of Newick string content. This has the highest priority.

Examples:

# Simple tree with branch lengths
(A:0.1,B:0.2,(C:0.3,D:0.4):0.5);

# Tree with internal node names
(A:0.1,B:0.2,(C:0.3,D:0.4)E:0.5)F;

# Tree with support values
(A:0.1,B:0.2,(C:0.3,D:0.4)100:0.5);

2. input.nwk.path

Description: Path to a Newick format tree file (.nwk, .newick, .tree).

Example file content:

# SpeciesTree.nwk
((Human:0.1,Chimp:0.08)Hominini:0.2,(Gorilla:0.15,Orangutan:0.25)GreatApes:0.3)Hominidae;

3. input.tableLike.path

Description: Path to a table-like tree file (Intuitive Tree Format, ITP).

This format is more readable for humans and easier to edit manually.

Format rules:

  • Tab (\t) separates parent from children
  • Comma (,) separates sibling nodes
  • Colon (:) separates node name from branch length (like Newick)
  • First line defines the root node
  • Leaf nodes have empty children (or just their name)

Example 1 - Simple tree without branch lengths:

root	A,B
A	C,D
B	E,F
C
D
E
F

This represents the tree: ((C,D)A,(E,F)B)root;

Example 2 - Tree with branch lengths:

root:0	A:0.1,B:0.2
A:0.1	C:0.3,D:0.4
B:0.2	E:0.15,F:0.25

D:0.4
E:0.15
F:0.25

Newick Format Options

nwk.format

Description: Specifies the Newick format type for parsing. Reference: ETE3/ETE4 format definitions.

Default: 0 (flexible with support values)

Note on Support Values: Bootstrap/support values should be in the 0-100 range (e.g., 90, 95, 100), not decimal format (0.0-1.0). For example, use 90 to represent 90% bootstrap support, not 0.9.

Format Leaf Node Internal Node Example
0 name:dist support:dist (A:0.1,(B:0.2,C:0.3)90:0.4);
1 name:dist name:dist (A:0.1,(B:0.2,C:0.3)NodeX:0.4);
2 name:dist (strict) support:dist (strict) Same as 0, but all fields required
3 name:dist (strict) name:dist (strict) Same as 1, but all fields required
4 name:dist - (none) (A:0.1,(B:0.2,C:0.3));
5 name:dist :dist only (A:0.1,(B:0.2,C:0.3):0.4);
6 name:dist name only (A:0.1,(B:0.2,C:0.3)NodeX);
7 name only name only (A,(B,C)NodeX);
8 name only - (none) (A,(B,C));
9 - (none) - (none) (,(,));

nwk.remove.whitespace

Description: Whether to remove whitespace (spaces, tabs, carriage returns, and linefeeds) from the Newick file before parsing.

Default: F (False)

Values: T (True) or F (False)

Use case: Set to T when your Newick file contains line breaks or formatting whitespace that should be ignored.
