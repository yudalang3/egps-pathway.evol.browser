现代化树形视图

解决的问题

绘制进化树的可视化图片，优势在于高度交互式。支持多种树形布局、颜色配置和图形标注，可导出高质量矢量图形。

快速开始：

所有操作都比较直观、易懂。请按照图形用户界面的指示进行操作。

  1. 点击示例按钮查看输入数据。
  2. 点击执行按钮查看结果。

图形界面中控制面板新特性：

  1. 现代化的图形界面，不是之前灰蒙蒙的状态。可记忆一些用户操作
  2. 控制面板按钮，分为四类：
    1. 第一类：手势按钮
    2. 第二类：正常按钮
    3. 第三类：会计数的手势按钮
    4. 第四类：Swing Toggle按钮
  3. 力推这个有苹果桌面风格的Toggle按钮，在Tree layout和Creative mode面板中有所应用
  4. VOICM解决参数爆炸：
    1. 可以记录的Bookmark书签，支持自由操作，导入导出
    2. 在输入面板处可以设置所有参数

输入参数参考

进化树输入方式

有三种方式可以输入进化树（按优先级排序）：

1. input.nwk.string

说明： 直接输入Newick格式字符串。优先级最高。

示例：

# 带分支长度的简单树
(A:0.1,B:0.2,(C:0.3,D:0.4):0.5);

# 带内部节点名称的树
(A:0.1,B:0.2,(C:0.3,D:0.4)E:0.5)F;

# 带支持度值的树
(A:0.1,B:0.2,(C:0.3,D:0.4)100:0.5);

2. input.nwk.path

说明： Newick格式树文件的路径（.nwk, .newick, .tree）。

文件内容示例：

# SpeciesTree.nwk
((Human:0.1,Chimp:0.08)Hominini:0.2,(Gorilla:0.15,Orangutan:0.25)GreatApes:0.3)Hominidae;

3. input.tableLike.path

说明： 表格式树文件的路径（直观树格式，ITP）。

这种格式对人类更易读，也更方便手动编辑。

格式规则：

  • 制表符（\t）分隔父节点和子节点
  • 逗号（,）分隔同级节点
  • 冒号（:）分隔节点名称和分支长度（与Newick格式相同）
  • 第一行定义根节点
  • 叶节点的子节点为空（或只有节点名称）

示例1 - 不带分支长度的简单树：

root	A,B
A	C,D
B	E,F
C
D
E
F

对应的Newick格式: ((C,D)A,(E,F)B)root;

B:0.2	E:0.15,F:0.25

Newick格式选项

nwk.format

说明： 指定解析时使用的Newick格式类型。参考：ETE3/ETE4格式定义。

默认值： 0（灵活格式，带支持度值）

支持度值说明： Bootstrap/支持度值应使用 0-100 范围（如：90, 95, 100）， 而非小数格式（0.0-1.0）。例如，使用 90 表示90%的bootstrap支持度，而不是 0.9。

格式 叶节点 内部节点 示例
0 名称:距离 支持度:距离 (A:0.1,(B:0.2,C:0.3)90:0.4);
1 名称:距离 名称:距离 (A:0.1,(B:0.2,C:0.3)NodeX:0.4);
2 名称:距离（严格） 支持度:距离（严格） 与0相同，但所有字段必填
3 名称:距离（严格） 名称:距离（严格） 与1相同，但所有字段必填
4 名称:距离 -（无） (A:0.1,(B:0.2,C:0.3));
5 名称:距离 仅:距离 (A:0.1,(B:0.2,C:0.3):0.4);
6 名称:距离 仅名称 (A:0.1,(B:0.2,C:0.3)NodeX);
7 仅名称 仅名称 (A,(B,C)NodeX);
8 仅名称 -（无） (A,(B,C));
9 -（无） -（无） (,(,));

nwk.remove.whitespace

说明： 是否在解析前移除Newick文件中的空白字符（空格、制表符、回车符和换行符）。

默认值： F（False，不移除）

可选值： T（True，移除）或 F（False，不移除）

使用场景： 当您的Newick文件包含换行或格式化空白需要忽略时，设置为T。
